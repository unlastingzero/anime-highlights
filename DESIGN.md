# 动漫精彩片段提取器 (Anime Highlight Extractor) - 设计方案

## 1. 系统目标
基于纯非 AI 启发式算法（Heuristics），在保证极高处理速度和零 GPU 成本的前提下，自动定位并截取动漫视频中最精彩的高潮片段（如激烈的战斗、情绪爆发点），并支持导出为高质量 MP4 或优化的动漫色板 GIF。

## 2. 核心打分维度 (Heuristics)
系统将视频划分为固定步长的“滑动窗口”（Sliding Window，例如窗口大小 3 秒，步长 1 秒），并为每个窗口计算以下三个维度的分数：

1.  **音频能量 (Audio Energy) - 权重 50%**
    *   **原理**：动漫的高潮绝大多数伴随着高燃 BGM、声优的嘶吼或强烈的爆炸音效。
    *   **实现**：提取音频流，转换为单声道并降采样（如 16kHz 以提升速度），计算该窗口内的 RMS (均方根) 能量峰值。
2.  **画面运动强度 (Video Dynamics) - 权重 30%**
    *   **原理**：战斗场面包含剧烈的像素级变化。
    *   **实现**：为了性能，不进行全视频光流计算。先通过音频和切分密度筛选出候选区间，再利用 OpenCV 对候选区间计算相邻帧的平均像素差 (Frame Difference)。
3.  **镜头切分密度 (Scene Cut Density) - 权重 20%**
    *   **原理**：高能打斗或紧张对峙时的分镜切换非常频繁。
    *   **实现**：利用 FFmpeg 的 `scenedetect` 滤镜，统计窗口内的硬切分（Hard Cut）次数。

## 3. 处理管线架构 (Pipeline Architecture)
系统采用“粗筛 -> 精筛 -> 对齐 -> 导出”的漏斗式架构，极大优化性能。

### Phase 1: 粗筛 (Coarse Filtering)
*   **动作**：使用 FFmpeg 极速提取整条音频并生成 `WAV`。并行运行 FFmpeg 提取全片的镜头切换点时间戳（Scene Changes）。
*   **计算**：按 3 秒滑动窗口计算**音频分数**和**镜头切分密度分数**。
*   **输出**：选取综合得分最高的前 Top-N 个区间（每个区间约 10-15 秒），作为候选高能片段。

### Phase 2: 精筛 (Fine Verification)
*   **动作**：仅针对这 Top-N 个短区间，使用 OpenCV 逐帧读取画面。
*   **计算**：计算区间的**画面运动强度分数**，并结合颜色对比度跳变（捕捉大招特效）。
*   **输出**：决出最终得分最高的 1 个区间（或由用户指定保留前几个）。

### Phase 3: 镜头边界对齐 (Boundary Alignment)
*   **动作**：直接截取固定时间（如 `[14:05, 14:15]`）会导致画面从动作进行到一半时突兀切入切出。
*   **计算**：以高分区间为核心，向前和向后寻找最近的**镜头硬切分点 (Scene Cut)**。
*   **输出**：动态调整后的截取时间戳（例如 `[14:03.2, 14:16.8]`），保证剪切的语义完整性。

### Phase 4: 高质量导出 (Export)
*   **MP4 导出**：使用 FFmpeg `-c:v copy`（如果关键帧允许）或高质量重编码 `-crf 18` 导出。
*   **GIF 导出**：针对动漫色彩鲜艳的特点，强制使用 `palettegen` 滤镜生成该片段专属的 256 色调色板，再通过 `paletteuse` 滤镜合成 GIF，彻底解决普通 GIF 工具生成的严重颗粒感。

## 4. 技术栈选型
*   **语言**：Python 3.10+
*   **核心引擎**：`ffmpeg-python` (调用 FFmpeg 的利器，速度远快于 MoviePy 原生处理，尤其在提取特征和高质量 GIF 生成时不可替代)。
*   **音频处理**：`librosa` (专业且稳定的音频特征提取) + `numpy`。
*   **视频处理**：`opencv-python` (仅用于精筛阶段的帧差计算，避免重度依赖)。
